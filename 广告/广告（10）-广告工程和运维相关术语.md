
# 广告系统工程与运维术语全景

## 一、竞价交易层（Bid & Transaction Layer）

| 术语 | 定义 | 说明 |
|------|------|------|
| **Bid Request** | 竞价请求 | ADX向DSP发送的竞价邀请 |
| **Bid Response** | 竞价响应 | DSP返回的出价信息 |
| **Bid ID** | 竞价ID | 单次竞价唯一标识 |
| **Impression ID** | 曝光ID | 广告展示唯一标识 |
| **Auction ID** | 拍卖ID | 竞价场次唯一标识 |
| **Seat** | 席位 | DSP在ADX的身份标识 |
| **Deal ID** | 交易ID | PDB/PA/PMP唯一标识 |
| **Price Floor** | 底价 | 媒体设置的最低成交价 |
| **Reserve Price** | 保留价 | 卖方最低接受价格 |
| **Clearing Price** | 清算价 | 实际成交价格 |
| **Second Price** | 第二高价 | 次高出价（GSP机制） |
| **First Price** | 第一高价 | 最高出价（FPB机制） |
| **Win Notice** | 赢标通知 | 通知DSP竞价成功 |
| **Loss Notice** | 输标通知 | 通知DSP竞价失败 |
| **Timeout** | 超时 | 竞价响应时限（通常100ms） |
| **No-Bid** | 不出价 | DSP选择不参与竞价 |
| **Bid Shading** | 出价优化 | 根据历史成交价动态降低出价 |
| **Bid Caching** | 出价缓存 | 复用历史出价降低延迟 |
| **Header Bidding** | 头部竞价 | 浏览器端多SSP并行竞价 |
| **Prebid** | Prebid框架 | 开源头部竞价解决方案 |
| **Open Auction** | 公开竞价 | 开放式RTB交易市场 |
| **Private Auction** | 私有竞价 | 邀请制竞价 |
| **PMP (Private Marketplace)** | 私有交易市场 | 邀请制程序化交易场所 |
| **PG (Programmatic Guaranteed)** | 程序化保量 | 保量保价的程序化交易 |
| **PA (Preferred Arrangement)** | 优先交易 | 优先于公开竞价的交易 |
| **PDB (Programmatic Direct Buy)** | 程序化直接购买 | 直接采购的程序化形式 |
| **RTB (Real-Time Bidding)** | 实时竞价 | 毫秒级竞价交易 |
| **OpenRTB** | OpenRTB协议 | IAB制定的竞价协议标准 |
| **Ads.txt** | 授权卖家列表 | 防域名欺诈，验证卖家身份 |
| **App-ads.txt** | 应用授权卖家列表 | 移动应用版本的ads.txt |
| **Sellers.json** | 卖家信息文件 | 公开SSP/DSP身份信息 |
| **Supply Path Optimization** | 供应路径优化 | 选择最优SSP链路降低成本 |
| **Demand Path Optimization** | 需求路径优化 | 优化DSP采购链路 |
