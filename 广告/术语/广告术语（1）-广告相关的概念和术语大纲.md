## 📚 广告技术术语大全大纲（按类别分类）

这里总结一下广告整个业务线相关的一些术语大纲，只做了一个粗分类。

---

### 一、【生态角色】
| 术语 | 定义 | 说明 |
|------|------|------|
| **Advertiser** | 广告主 | 出钱投放广告的企业或个人 |
| **Publisher** | 媒体 / 发布商 | 拥有流量的网站、App 开发者 |
| **DSP (Demand-Side Platform)** | 需求方平台 | 代表广告主自动购买广告流量的平台 |
| **SSP (Supply-Side Platform)** | 供应方平台 | 代表媒体自动售卖广告流量的平台 |
| **ADX (Ad Exchange)** | 广告交易平台 | 撮合 DSP 与 SSP 的中立交易市场 |
| **DMP (Data Management Platform)** | 数据管理平台 | 存储用户标签、提供人群定向能力 |
| **TD (Trading Desk)** | 程序化交易台 | 为大型广告主统一管理多个 DSP |
| **3PV (Third-Party Verification)** | 第三方验证 | 如 IAS、DoubleVerify，用于反作弊与品牌安全 |
| **Agency（广告代理公司）** | 代表广告主策划、执行广告活动的专业服务商 | 提供策略、创意、媒介购买等服务；大型代理公司如 WPP、阳狮、电通等常下设程序化交易台（Trading Desk） |
| **Creative Management Platform (CMP)** | 创意管理平台 | 帮助广告主批量生成、测试和优化动态广告素材（如 Banner、视频） |
| **Ad Server（广告服务器）** | 广告投放与追踪系统 | 用于控制广告展示逻辑、频次、归因等；分为买方（Buy-side Ad Server，如 Google Campaign Manager 360）和卖方（Sell-side Ad Server，如 Google Ad Manager） |
| **Attribution Provider（归因服务商）** | 归因分析平台 | 衡量用户转化路径中各触点贡献（如 AppsFlyer、Adjust、Branch、Kochava） |
| **Identity Resolution Provider（身份解析服务商）** | 身份识别与跨设备匹配服务 | 在 Cookie 限制背景下，提供基于邮箱、设备 ID 等的用户身份图谱（如 LiveRamp、The Trade Desk 的 Unified ID 2.0、Zeotap） |
| **Measurement & Analytics Platform（测量与分析平台）** | 效果评估与数据洞察工具 | 如 Nielsen、Comscore、Google Analytics，用于衡量曝光、到达率、品牌提升等 |
| **Consent Management Platform (CMP)** | 同意管理平台 | 根据 GDPR/CCPA 等法规收集用户隐私同意（如 OneTrust、Sourcepoint） |
| **Header Bidding Wrapper（首价竞价封装器）** | 首价竞价技术组件 | 帮助发布商并行调用多个 SSP/DSP 实现公平竞价（如 Prebid.js） |
| **Contextual Targeting Provider（上下文定向服务商）** | 基于页面内容而非用户行为的定向技术 | 在隐私合规趋势下重新兴起（如 GumGum、Criteo Contextual） |
| **Ad Fraud Detection Vendor（广告反欺诈厂商）** | 专门检测虚假流量与点击欺诈 | 部分与 3PV 重叠，但也有独立方案（如 White Ops → HUMAN, Spider.io） |


---

### 二、【交易模式】
| 术语 | 定义 | 说明 |
|------|------|------|
| **RTB (Real-Time Bidding)** | 实时竞价 | 每次曝光机会进行毫秒级拍卖 |
| **RTA (Real-Time API)** | 实时 API | 媒体主动调用 DSP 接口决定是否投放 |
| **Open Auction (OA)** | 公开竞价 | 所有 DSP 可参与的标准 RTB |
| **Private Auction (PA)** | 私有竞价 | 媒体邀请特定 DSP 参与的竞价 |
| **Preferred Deal (PD)** | 优先交易 | 固定价格、非保量，DSP 可选择是否买 |
| **Programmatic Guaranteed (PG)** | 程序化保量 | 固定价格 + 保量，类似直投 |
| **Direct Deal** | 直接交易 | 广告主与媒体线下签约，不走程序化 |
| **Waterfall** | 瀑布流 | 按优先级顺序依次请求广告源 |
| **Header Bidding** | 前置竞价 | 并行向多个 SSP 请求，价高者得 |
| **Contextual Targeting** | 上下文定向 | 基于页面/App 内容投放（无需用户 ID） |
| **Audience Targeting** | 人群定向 | 基于 DMP 用户标签投放 |
| **Geotargeting / Geofencing** | 地域定向 / 地理围栏 | 基于地理位置触发广告 |

---

### 三、【协议与标准】
| 术语 | 定义 | 说明 |
|------|------|------|
| **OpenRTB** | 开放实时竞价协议 | IAB 制定的 RTB 通信标准（v2.5 最常用） |
| **VAST** | Video Ad Serving Template | 视频广告请求/返回标准（XML） |
| **VMAP** | Video Multiple Ad Playlist | 控制多段视频广告播放顺序 |
| **VPAID** | Video Player-Ad Interface Definition | 旧版互动视频广告接口（已淘汰） |
| **SIMID** | Secure Interactive Media Interface Definition | VPAID 继任者，更安全 |
| **MRAID** | Mobile Rich Media Ad Interface Definitions | 移动端富媒体广告标准 |
| **Ads.txt / App-ads.txt** | 广告授权文件 | 媒体声明合法 SSP 列表，防域名盗用 |
| **Seller.json** | 卖家身份声明 | 配合 Ads.txt 使用 |
| **OpenRTB SupplyChain Object** | 流量来源透明化对象 | 防止流量劫持 |
| **TCF (Transparency & Consent Framework)** | 用户授权框架 | GDPR 合规标准 |

---

### 四、【出价与优化策略】
| 术语 | 定义 | 说明 |
|------|------|------|
| **CPM / CPC / CPA** | 千次展示 / 点击 / 转化成本 | 基础计费方式 |
| **oCPM / oCPC / oCPA** | 目标转化出价 | eCPM = pCVR × target_CPA |
| **oCPX** | 智能出价统称 | 国内对 oCPM/oCPA 的泛称 |
| **tCPA / tROAS** | 目标 CPA / 目标广告支出回报率 | Google/Meta 术语 |
| **Bid Shading** | 出价遮蔽 | 在第二价格拍卖中降低出价以节省成本 |
| **Budget Pacing** | 预算平滑 | 控制预算在周期内均匀/智能消耗 |
| **Frequency Capping** | 频次控制 | 限制用户单位时间曝光次数 |
| **Recency Capping** | 近期频控 | 如“7天内最多3次” |
| **Bid Multiplier** | 出价系数 | 对特定人群/时段加价 |
| **Auto Bidding** | 自动出价 | 系统自动调价达成目标 |
| **CBO (Campaign Budget Optimization)** | 计划预算优化 | 在广告组间动态分配预算 |
| **Lookalike Audience** | 相似人群扩展 | 基于种子用户找相似人群 |
| **Creative Fatigue** | 创意疲劳 | 同一创意反复曝光导致 CTR 下降 |
| **Cold Start Problem** | 冷启动问题 | 新计划无历史数据，模型无法预估 |

---

### 五、【归因与测量】
| 术语 | 定义 | 说明 |
|------|------|------|
| **Impression (Imp)** | 曝光 | 广告被展示一次 |
| **Click-Through Conversion (CTC)** | 点击后转化 | 用户点击广告后完成转化 |
| **View-Through Conversion (VTC)** | 曝光后转化 | 未点击但后续转化 |
| **Attribution Window** | 归因窗口 | 如 click_7d_view_1d |
| **Last Click / First Click / Linear / Time Decay** | 归因模型 | 分配转化功劳的方式 |
| **Shapley Value Attribution** | 沙普利值归因 | 基于博弈论的公平分配 |
| **Incrementality** | 增量效果 | 广告带来的真实新增转化 |
| **Ghost Ads** | 幽灵广告 | 对照组看到空白广告位，用于增量测试 |
| **MMM (Marketing Mix Modeling)** | 营销组合建模 | 宏观渠道效果评估 |
| **Conversion API (CAPI)** | 转化回传 API | 服务器端上报转化（替代 Pixel） |
| **Cross-Device Attribution** | 跨设备归因 | 手机 → PC 转化归因 |
| **Offline Conversion** | 离线转化 | 门店扫码、电话销售等回传 |
| **Store Visit Attribution** | 到店归因 | 基于 GPS 判断是否到店 |

---

### 六、【隐私与合规】
| 术语 | 定义 | 说明 |
|------|------|------|
| **GDPR / CCPA / PIPL** | 隐私法规 | 欧盟、加州、中国个人信息保护法 |
| **Consent Management** | 用户授权管理 | 通过 CMP 弹窗获取同意 |
| **IDFA / AAID / OAID** | 设备标识符 | iOS / Android / 中国安卓匿名 ID |
| **ATT (App Tracking Transparency)** | 应用跟踪透明度 | iOS 14+ 弹窗授权机制 |
| **SKAdNetwork (SKAN)** | 苹果归因框架 | iOS 隐私安全归因（v2/v3/v4） |
| **Conversion Value Schema** | 转化值编码规则 | SKAN 中 6-bit 值的业务含义 |
| **Probabilistic ID** | 概率 ID | 基于 IP+UA 生成的匿名 ID |
| **Deterministic ID** | 确定性 ID | 登录态用户 ID（如微信 openid）|
| **Privacy Sandbox** | 谷歌隐私沙盒 | Cookie 替代方案（Topics, FLEDGE）|
| **FLEDGE / Protected Audience** | 隐私保护重定向 | 在本地完成重定向逻辑 |
| **Topics API** | 兴趣群组 API | 谷歌提出的上下文+群组定向 |

---

### 七、【反作弊与质量】
| 术语 | 定义 | 说明 |
|------|------|------|
| **IVT (Invalid Traffic)** | 无效流量 | 总称 |
| **GIVT (General IVT)** | 一般无效流量 | 已知爬虫、蜘蛛 |
| **SIVT (Sophisticated IVT)** | 高级无效流量 | 设备农场、模拟器 |
| **Device Fingerprinting** | 设备指纹 | Canvas/WebGL/电池等特征 |
| **Click Spamming / Injection** | 点击劫持 | 尤其 Android 渠道常见 |
| **SDK Spoofing** | SDK 伪造 | 伪造广告请求刷量 |
| **Bot Detection** | 机器人检测 | 行为序列异常识别 |
| **Brand Safety** | 品牌安全 | 避免出现在不良内容旁 |
| **Suitable Inventory** | 合规流量池 | 媒体提供的“干净”流量 |
| **Ad Fraud Postback** | 作弊回扫 | 事后申请退款或抵扣 |

---

### 八、【工程与运维】
| 术语 | 定义 | 说明 |
|------|------|------|
| **Bid Request / Bid Response** | 竞价请求/响应 | OpenRTB 核心消息 |
| **Seat** | 席位 | DSP 在 ADX 中的身份标识 |
| **Deal ID** | 交易 ID | PDB/PA 唯一标识 |
| **Price Floor / Reserve Price** | 底价 | 媒体设置的最低成交价 |
| **Win Notice** | 赢标通知 | 用于计费对账 |
| **Billable Event** | 可计费事件 | 曝光 or 点击？需对齐 |
| **Reconciliation** | 对账 | DSP 与 SSP 核对数据 |
| **Discrepancy** | 数据差异 | 通常允许 ±5% |
| **Makegood** | 补量 | 因未达标而免费补投 |
| **Log Sampling** | 日志采样 | 高 QPS 下降本策略 |
| **Shadow Mode** | 影子模式 | 新策略上线前对比测试 |
| **Canary Release** | 灰度发布 | 逐步放量新版本 |
| **Cost per Millisecond** | 毫秒成本 | 延迟每增 10ms，收入可能降 1% |
| **Geo-Distributed Bidding** | 地理分布式出价 | 靠近 ADX 部署降低延迟 |
| **Online Learning** | 在线学习 | 实时更新 pCTR/pCVR 模型 |
| **Feature Store** | 特征存储 | 供模型和规则引擎使用 |

---

### 九、【国内特色术语】
| 术语 | 含义 |
|------|------|
| **放量 / 稳投 / 跑飞** | 出价策略状态 |
| **冷启 / 热启** | 广告计划阶段 |
| **建模成功** | oCPX 积累足够转化，模型生效 |
| **掉量** | 曝光量突然下降 |
| **穿山甲 / 优量汇 / Uni Desk** | 字节 / 腾讯 / 阿里 DSP |
| **联盟流量** | 第三方 App 聚合流量 |
| **直投 vs 联盟** | 自有 App vs 第三方 App |
| **应用直达 / 唤醒广告** | DeepLink 拉活已安装 App |
